# 使用手冊

> 本手冊面向**終端使用者**。撰寫中,待 MVP 功能成形後補齊畫面截圖與步驟。

## 適用影片

ZebraGuard 僅支援**行車記錄器(dashcam)**影片。固定式監視器 / 路口 CCTV 目前不在支援範圍內。

## 規劃內容

- [ ] 安裝與首次啟動
- [ ] 建立新專案:匯入行車記錄器影片
- [ ] 開始分析:GPU / CPU 模式差異、預估時間、首次下載斑馬線分割模型
- [ ] 審查違規候選:採用 / 拒絕 / 調整時段、騎士與自車為何被排除
- [ ] 匯出檢舉輔助包
- [ ] 如何使用輔助包向警政署提報
- [ ] 常見問題
  - Defender / SmartScreen 警告怎麼辦
  - 分析很慢怎麼辦 / 為什麼 CPU 模式要幾小時
  - 為什麼我的車牌/行人沒被偵測到
  - 為什麼同一台車被連續報兩次違規(停讓後起步)
  - 為什麼我的自車 / 前車被標示為違規
  - 檔案存在哪裡
  - 怎麼刪除我的資料

## 撰寫原則

- 用語以使用者角度,避免技術名詞(例如:Mask2Former → 稱「斑馬線自動辨識」;ByteTrack → 稱「追蹤」)
- 截圖優先於文字
- 每節目標限於 3 分鐘內可看完
- 明確區分「必要步驟」與「進階調整」
- 使用者不需要繪製 ROI 或校正 homography — 這些步驟已被自動斑馬線辨識取代
